# 약관·개인정보처리방침 유지 규칙 (영구)

> 기능 개발·배포 시 약관(`docs/TERMS_{ja,kr}.md`)과 개인정보처리방침(`docs/PRIVACY_{ja,kr}.md`)이 실제 서비스 동작과 일치하도록 유지하기 위한 영구 규칙.

## 원칙

- 약관·개인정보처리방침은 **실제 배포된 기능**만 반영한다 (미구현 기능 선제 반영 금지)
- 한국어(`_kr.md`)와 일본어(`_ja.md`)는 항상 **동시에** 수정한다
- 개정 시 문서 상단 "최종 갱신일" · "시행일"을 업데이트한다
- PIPA(한국) + APPI(일본) 이중 준수 관점으로 검토한다

## 기능 변경 시 자동 체크

아래 변경을 건드리면 반드시 `/약관확인` 실행 또는 수동 영향 검토:

- **Auth 변경** (회원가입 필드 추가/삭제, 소셜 로그인, MFA, 본인확인) → PRIVACY 수집항목·처리목적
- **DB 스키마 신규 컬럼** (특히 `influencers`, `applications`, `deliverables`, `receipts`(Stage 7에서 deliverables로 통합 예정)) → PRIVACY 수집하는 개인정보
- **외부 서비스 추가** (새 SMTP, 결제 PG, SNS API, 분석 SDK) → PRIVACY 처리위탁·제3자 제공
- **결제·정산·세무** (원천징수율, 환율, 수수료 변경) → TERMS 정산 조항
- **관리자 열람 범위 확대** (새 열람 권한, 감사 로그) → PRIVACY 접근권한
- **마케팅/알림** (푸시, 뉴스레터, SMS) → PRIVACY 마케팅 동의
- **쿠키·로컬스토리지** (분석, 세션 저장) → PRIVACY 자동수집
- **국외이전 경로 변경** (Supabase 리전 이전, 새 해외 벤더) → PRIVACY 국외이전

## 배포 시점 점검

- 운영 배포(main merge) 후 `/약관확인`로 누락 여부 확인
- 약관 변경이 **중대 변경**(수집항목·제3자 제공·국외이전 추가, 정산 규칙 변경)이면:
  - 기존 사용자에게 **재동의** 또는 **사전 통지**(7일 이상) 필요
  - 앱 내 공지 + 메일 발송 계획 함께 제안

## 에이전트 위임

- 약관/정책 관련 작업은 `/약관확인` 명령을 우선 호출
- `reverb-planner`는 기능 기획 단계에서 "약관 영향 있음" 여부를 체크리스트에 포함
- `reverb-reviewer`는 commit 직전 DB 스키마·Auth·외부 서비스 변경 시 "약관 반영 필요" 경고
