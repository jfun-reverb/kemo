# 사용자 인터랙션 규칙 (영구)

> 이 파일의 모든 항목은 **모든 세션/모든 에이전트에 영구 적용**되는 규칙이다. 세션 한정 메모리가 아니다. 변경 시 전 에이전트 재교육 필요.

## 선택지 질문은 반드시 `AskUserQuestion` 도구 사용
- 2개 이상 선택지가 등장하는 순간 **즉시 `AskUserQuestion` 도구 호출** — 본문에 "1. A / 2. B / 3. C" 텍스트로 나열해서 타이핑 답변 유도 금지
- 추천 옵션은 맨 앞에 배치하고 라벨 끝에 `(추천)` 표기
- 질문 본문·헤더·옵션 라벨·설명 **전부 한국어**
- 헤더는 12자 이내
- 옵션은 2~4개, 각 옵션에 설명 1~2문장
- 예/아니오 단순 확인도 원칙적으로 `AskUserQuestion` 사용 (본문 한 줄짜리 "진행할까요?"만 예외)
- **서브에이전트가 텍스트로 Q1~Q6을 반환해도, 메인 Claude는 반드시 `AskUserQuestion`으로 변환해서 사용자에게 전달** — 서브에이전트 출력을 그대로 붙여넣고 "골라주세요"라고 하면 규칙 위반

**Why:** 사용자는 외출 중 핸드폰으로 답변하는 경우가 많아 키보드 선택 UI가 훨씬 빠름. 텍스트 나열 방식은 타이핑 강제 → 반복 지적 유발.

## 응답 언어
- 모든 응답은 한국어. 일본어/영어로 대답 금지
- 관리자 페이지 관련 질문에도 답변은 한국어 (페이지 UI만 한국어, 인플루언서 UI만 일본어)

## 🚨 약어·전문용어 자제 (의무 — 사용자는 비개발자)

**원칙**: 사용자는 개발자가 아니므로 영어 약어와 전문용어를 풀어써야 한다. 2026-05-04 사용자 명시 지적: "FK나 R/S 같은 약어는 모르겠다, 한글로 알려달라."

### 단독 사용 금지 (반드시 한글 풀이 또는 한글+괄호 병기)
- **FK** → "외래 키" 또는 "다른 표의 행을 가리키는 연결"
- **RPC** → "원격 호출 함수" 또는 "데이터베이스에 미리 만든 명령 함수"
- **RLS** → "행 단위 보안 정책" / "사용자별 데이터 접근 제어"
- **RBAC** → "역할 기반 권한 제어"
- **R/S** (REVERB) → "리뷰어/시딩 폼 타입"
- **DDL/DML** → "데이터베이스 구조 변경 / 데이터 변경 명령"
- **XSS/CSRF/SSRF/OWASP** → 한글 + 괄호 병기 ("교차 사이트 스크립팅(XSS)" 등)

### 처음 등장 시 한글 + 괄호 약어
- "외래 키(FK)" 처럼 첫 등장만 병기, 같은 답변 내 반복은 약어 또는 한글 모두 OK

### 자릿수 표시 약어
- NNNN → "4자리 숫자", ### → "3자리 숫자"로 풀어쓰기

### 일반 약어는 풀어쓰지 않아도 됨
- DB, UI, URL, IP, PC, OS, CSS, HTML, JS — 보편적이라 OK
- 사용자가 새로 모른다고 지적하면 본 섹션에 추가

### 서브 에이전트 결과 검열
- planner/supabase-expert/reviewer/qa-tester 등 서브 에이전트 답변에 약어가 그대로 있으면 메인 Claude가 사용자에게 전달 **전에 한글로 재가공**
- 모든 서브 에이전트도 본 규칙 준수 의무 (이 파일은 모든 에이전트가 읽음)

### AskUserQuestion 옵션 검열
- 옵션 label·description 모두 약어 대신 한글 풀이
- 옵션 작성 후 호출 직전 자체 grep 권장

### 반복 위반 시
- `memory/feedback_no_jargon.md` 상단에 날짜 + 위반 사례 누적
- 사용자가 다시 지적하면 신뢰도 훼손

## 🚨 한국어 오탈자 자체 점검 (의무 — 반복 위반 금지)

**문서·노션·HTML 등 한국어 텍스트 대량 작업 완료 직전 반드시 아래 순서로 자체 점검:**

### 1단계: 로컬 파일 grep
```
Grep("(캐페인|캐웩인|캠패인|켐페인|행버거|재생각|부와다니|마카일 기록|돿습|됏|뗴다가|뮤티|팔로움|컴럼|바뀍니다|뜨:)", output_mode="content")
```

### 2단계: 노션 전수 점검
- 수정한 모든 노션 페이지를 `mcp__claude_ai_Notion__notion-fetch`로 다시 가져와 본문 육안 스캔
- 표(`<table>`) 내부, FAQ 자유 서술, 주의 박스(`> ⚠️`, `> 💡`)는 특히 꼼꼼히
- `mcp__claude_ai_Notion__notion-search`로 `캐페인`, `캐웩`, `현장 없음`, `뜨:` 등 의심 키워드 검색

### 확정 오탈자 리스트 (memory/feedback_korean_typos.md와 동기화)
- **캠페인** ↔ 캐페인/캐웩인 (2회 반복 지적, 노션 MCP 유니코드 혼동 주의: `\ucea0\ud398\uc778`)
- **햄버거** ↔ 행버거
- **현재** ↔ 현장 (문맥 혼동)
- **뜸** ↔ 뜨 (종결형 불완전)
- **바뀝니다** ↔ 바뀍니다
- **제거됐습니다** ↔ 제거돿습니다
- **콘텐츠** ↔ 컨텐츠
- **메시지** ↔ 메세지

### 사용자가 오탈자 지적 시
- 단순 하나만 고치지 말고 **해당 세션의 모든 작업물 전수 재점검**
- 발견된 오탈자는 `~/.claude/projects/-Users-younggeunkim-Documents-projects-reverb-jp/memory/feedback_korean_typos.md`에 누적 추가

**Why:** 사용자가 2회 이상 지적한 항목을 또 위반하면 신뢰 훼손. 노션은 MCP 유니코드 이스케이프 버그로 ghost 오탈자 위험 특히 높음. 작업 완료 시그널 내기 전 반드시 점검.

## 변경 프로세스
- 개선 요청 시: **계획 먼저 설명 → 영향 체크 → 애매한 부분 질문** → 승인 후 구현
- "애매한 부분 질문"도 위 `AskUserQuestion` 규칙 적용

## 규칙 영속화
- 사용자가 "이렇게 해줘", "저렇게 해줘"로 지시한 내용은 세션 한정이 아닌 **영구 규칙**으로 간주
- `.claude/rules/*.md`에 기록 (이 파일 또는 도메인별 파일)
- 동시에 `~/.claude/projects/.../memory/`에 feedback 메모리로도 저장 (이중 안전망)
- 서브에이전트 동작과 관련된 규칙이면 `.claude/agents/*.md`에도 반영

## 배포 전 필수 에이전트 호출 (스킵 금지)

**모든 `git commit` 직전에 반드시 수행 (단순 한 줄 오탈자 예외 외 스킵 금지):**

```
[필수 체크]
□ reverb-reviewer 호출 완료
□ Supabase/Auth/RLS/마이그레이션 변경 있으면 → reverb-supabase-expert 호출 완료
□ 인증/신청/관리자 플로우 변경 또는 운영 배포 직전 → reverb-qa-tester 호출 완료
□ 기능 추가·개선·리팩토링 시작 전 (규모 무관) → reverb-planner 호출 완료
```

- "한 줄짜리 간단한 수정이니까" 핑계로 reviewer 스킵 금지
- **스스로 판단해서 스킵하면 반드시 재지적받는다** (이전 세션에서 반복 위반 사례 있음)
- 메인 Claude가 push/merge 누르기 전 위 체크리스트를 텍스트로 되뇌고 확인
- `.claude/rules/git.md`의 "에이전트 호출 의무" 섹션과 동일한 내용이지만, 반복 위반으로 인해 이 파일에도 중복 명시 (이중 안전망)

**Why:** 이전 세션·이번 세션에서 reviewer 호출 누락 사례 반복 발생. "작은 변경이니까"가 가장 흔한 스킵 핑계 — 그러나 작은 변경도 stale 참조/z-index 충돌/RLS 누락 등을 만들 수 있으므로 예외 없이 호출.
