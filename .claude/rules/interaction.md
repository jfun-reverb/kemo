# 사용자 인터랙션 규칙 (영구)

> 이 파일의 모든 항목은 **모든 세션/모든 에이전트에 영구 적용**되는 규칙이다. 세션 한정 메모리가 아니다. 변경 시 전 에이전트 재교육 필요.

## 선택지 질문은 반드시 `AskUserQuestion` 도구 사용
- 2개 이상 선택지가 등장하는 순간 **즉시 `AskUserQuestion` 도구 호출** — 본문에 "1. A / 2. B / 3. C" 텍스트로 나열해서 타이핑 답변 유도 금지
- 추천 옵션은 맨 앞에 배치하고 라벨 끝에 `(추천)` 표기
- 질문 본문·헤더·옵션 라벨·설명 **전부 한국어**
- 헤더는 12자 이내
- 옵션은 2~4개, 각 옵션에 설명 1~2문장
- 예/아니오 단순 확인도 원칙적으로 `AskUserQuestion` 사용 (본문 한 줄짜리 "진행할까요?"만 예외)
- **서브에이전트가 텍스트로 Q1~Q6을 반환해도, 메인 Claude는 반드시 `AskUserQuestion`으로 변환해서 사용자에게 전달** — 서브에이전트 출력을 그대로 붙여넣고 "골라주세요"라고 하면 규칙 위반

**Why:** 사용자는 외출 중 핸드폰으로 답변하는 경우가 많아 키보드 선택 UI가 훨씬 빠름. 텍스트 나열 방식은 타이핑 강제 → 반복 지적 유발.

## 응답 언어
- 모든 응답은 한국어. 일본어/영어로 대답 금지
- 관리자 페이지 관련 질문에도 답변은 한국어 (페이지 UI만 한국어, 인플루언서 UI만 일본어)

## 변경 프로세스
- 개선 요청 시: **계획 먼저 설명 → 영향 체크 → 애매한 부분 질문** → 승인 후 구현
- "애매한 부분 질문"도 위 `AskUserQuestion` 규칙 적용

## 규칙 영속화
- 사용자가 "이렇게 해줘", "저렇게 해줘"로 지시한 내용은 세션 한정이 아닌 **영구 규칙**으로 간주
- `.claude/rules/*.md`에 기록 (이 파일 또는 도메인별 파일)
- 동시에 `~/.claude/projects/.../memory/`에 feedback 메모리로도 저장 (이중 안전망)
- 서브에이전트 동작과 관련된 규칙이면 `.claude/agents/*.md`에도 반영
