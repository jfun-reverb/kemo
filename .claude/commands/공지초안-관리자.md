# 공지초안-관리자

REVERB JP 관리자 페이지의 `/admin/#admin-notices` 에 등록할 공지 초안을 **Markdown + HTML 2종 모두** 자동 작성합니다. 메인 Claude 는 이 스킬을 호출받으면 아래 절차로 진행합니다.

> 🚨 **2026-04-22 중요 정정**: 과거 이 문서에 "Quill 에디터가 Markdown paste 를 자동 변환한다"고 적혀 있었으나 **사실이 아님**. 실제 `dev/js/admin.js:5693` 의 공지 에디터는 Quill 2.0.2 vanilla 에 `bold/italic/underline/strike/list/link/blockquote` 툴바만 구성 — **MD 플러그인 없음**. 마크다운을 그대로 붙여넣으면 `**굵게**`·`-` 리스트가 literal 텍스트로 찍힙니다.
>
> **올바른 경로 2가지**:
> 1. Quill 툴바 **`<>` HTML source 토글** → HTML 버전을 그대로 붙여넣기 → 다시 토글 해제 (권장)
> 2. Quill visual 영역에 **HTML 코드를 직접 paste** (HTML matchVisual=false 로 일부 형식 보존)
>
> MD 버전은 **Notion/Slack/PR 설명** 등 외부 공유용으로만 사용하세요.

## 입력 해석

사용자 인자(`$ARGUMENTS`) 패턴:

1. **인자 없음** → 최근 `main` 머지 커밋을 자동 탐지
   - `git log origin/main --merges --oneline -1` 로 마지막 merge 커밋 해시 추출
   - `git log <merge>^..<merge>` 로 포함된 feature 커밋들의 메시지 수집
2. **`PR <번호>`** → `gh pr view <번호> --json title,body,commits` 로 PR 정보 수집
3. **`warning <짧은 설명>`** → 긴급 경고 템플릿으로 바로 작성
4. **`system_update|release|warning|general <자유 설명>`** → 해당 카테고리로 작성

## 카테고리 자동 분류 휴리스틱

커밋 메시지/파일 경로를 보고 제안:

- `system_update` — `supabase/migrations/`, `dev/js/admin.js`, `dev/admin/`, `retryWithRefresh`, RLS/RPC 키워드, 관리자 내부 도구 개선
- `release` — `dev/index.html`, `dev/js/campaign.js`, `dev/lib/i18n/`, 인플루언서 UI/일본어 변경, 광고주 신청 폼 변경
- `warning` — `chore: trigger redeploy`, `hotfix`, `revert`, `Brevo`, `rate limit`, 장애 키워드
- `general` — 위에 해당 없는 경우

복수 성격이면 가장 큰 카테고리 1개 선택하고 본문에 보조 변경사항 기재.

````
## 📣 공지 초안

**카테고리**: system_update (시스템 업데이트)
**제목**: 인플루언서 상세 모달 통합 개편
**상단 고정**: ☐

### 📄 Markdown 버전 (Notion/Slack/PR 설명용 — 외부 공유)

인플루언서 관리자 상세 화면이 모달 기반으로 재구성됐습니다.

- 이름 옆 **인증/위반/블랙리스트 배지** 전역 노출
- **상태 관리 + 관리자 이력** 한 카드로 통합

### 🌐 HTML 버전 (관리자 페이지 `/admin#admin-notices` Quill 에디터 등록용 — 필수)

```html
<h3>인플루언서 상세 모달 통합 개편</h3>
<p>인플루언서 관리자 상세 화면이 모달 기반으로 재구성됐습니다.</p>
<ul>
  <li>이름 옆 <strong>인증/위반/블랙리스트 배지</strong> 전역 노출</li>
  <li><strong>상태 관리 + 관리자 이력</strong> 한 카드로 통합</li>
  <li>위반 기록에 <strong>증빙 파일 첨부</strong> + 라이트박스 확대</li>
  <li>목록 필터·검색 상단 일관화</li>
</ul>
<p><strong>영향 범위</strong>: 관리자 페이지 한정.</p>
```

### 근거 (생성 시 참고한 커밋)
- 3a1655b feat(admin): influencer verify · violation · blacklist management
- 88d6d3e feat(admin): violation evidence uploads + ESC close
````

동시에 **두 개 파일로 저장**:
- `docs/admin-notices/{YYYY-MM-DD}-{category}-{slug}.md` — Markdown (외부 공유용)
- `docs/admin-notices/{YYYY-MM-DD}-{category}-{slug}.html` — **Quill 호환 HTML** (관리자 페이지 등록용, 필수)

`docs/admin-notices/` 폴더가 없으면 생성. slug 는 제목을 소문자·하이픈 변환.

## 작성 원칙

- **title**: 1줄, 35자 이내, 한국어. "무엇이 바뀌었나" 요약
- **HTML 버전 제한** (실제 Quill 2.0.2 toolbar 매칭):
  - 허용: `<h2>`/`<h3>`/`<h4>`, `<p>`, `<strong>`, `<em>`, `<u>`, `<s>`, `<ul>`/`<ol>`/`<li>`, `<a>`, `<blockquote>`, `<br>`
  - **금지**: `<table>` (바닐라 Quill은 테이블 미지원 — 리스트로 변환), `<hr>`, `<img>`, `<script>`, `<iframe>`, `<style>`, 코드블록
- **표 대체**: 표 필요 시 `<ol>` 순번 리스트 또는 `<p><strong>항목</strong>: 내용</p>` 나열로
- **영향 범위 명시**: "관리자 페이지 한정" / "인플루언서 앱만" / "운영서버만 배포됨" 등

## 제한·안내

- 이 커맨드는 **초안만 생성**. 실제 DB INSERT 는 사용자가 관리자 페이지에서 수동으로 수행
- 직접 SQL 로 INSERT 하고 싶으면 사용자가 요청 시 `supabase/patches/YYYY-MM-DD_admin_notice.sql` 파일 추가 생성
- 확인 없이 자동 등록 금지 — 운영 DB 에 영향
- **Quill 붙여넣기 필수 경로**: 관리자 페이지 `/admin#admin-notices` 의 공지 작성 모달에서 **툴바의 `<>` (HTML source) 토글**을 누른 후 위 HTML 버전을 그대로 붙여넣기. 다시 토글 해제하면 서식이 적용된 상태로 보임. 마크다운 원문을 그대로 붙여넣으면 `**굵게**`·`-` 리스트가 literal 텍스트로 찍힘 (2026-04-22 실사용자 피드백)

## 작업 흐름

1. 인자 해석 → 커밋/PR/키워드 수집
2. 카테고리 자동 분류
3. 제목·본문 Markdown 작성 (위 원칙 준수)
4. `docs/admin-notices/` 폴더 없으면 `mkdir -p` 로 생성
5. 파일 저장 + 메시지에 전체 표시
6. 마지막에 "📋 관리자 페이지에 등록하려면: /admin/#admin-notices → 새 공지 작성 → 위 제목·본문(Markdown) 복사·붙여넣기. 변환이 깨지면 Quill 에서 `Cmd+Shift+V`(서식 없이 붙여넣기) 또는 HTML 버전 요청" 안내
