# /약관확인 — 약관·개인정보처리방침 영향 점검

운영 배포 이후 코드 변경이 약관(`docs/TERMS_{ja,kr}.md`) 또는 개인정보처리방침(`docs/PRIVACY_{ja,kr}.md`)에 영향을 주는지 점검하고 수정안을 제안한다.

## 실행 절차

1. **변경 범위 파악**
   - 인자 없으면 기본 `main` 브랜치 기준 최근 20개 커밋 분석
   - 인자로 범위 지정 가능 (예: `/약관확인 v1.2..main`, `/약관확인 커밋수=10`)
   - `git log --oneline <range>`, `git diff <range> --stat`, 주요 파일 `git diff` 확인

2. **영향 매트릭스 적용**
   각 변경을 아래 카테고리에 매핑:

   | 변경 유형 | 영향 문서 | 체크 항목 |
   |---|---|---|
   | Auth (회원가입/로그인/소셜/MFA) | PRIVACY | 수집항목, 본인확인 절차, 처리목적 |
   | DB 스키마 신규 컬럼 (influencers, applications 등) | PRIVACY | 수집하는 개인정보 목록, 보유기간 |
   | 결제/정산/원천징수/환율 | TERMS | §정산, §세무, 수수료 |
   | 외부 서비스 추가 (Brevo/Supabase/Vercel/PayPal/PortOne/SNS API) | PRIVACY | 처리위탁·제3자 제공 항목, 국외이전 |
   | SMTP/알림/마케팅 동의 | PRIVACY | 마케팅 수신, 광고성 정보 |
   | 관리자 권한·열람·감사 로그 | PRIVACY | 접근권한, 열람자, 로그 보관 |
   | 콘텐츠 업로드·검수·PR 태그 (스테스 규제) | TERMS | §콘텐츠 의무, §PR 표시 |
   | 해외 결제·환율·국제 배송 | TERMS + PRIVACY | 국외이전, 통관 정보 |
   | 캠페인 취소·환불·분쟁 | TERMS | §취소·환불, §분쟁해결 |
   | 쿠키/로컬스토리지/분석 SDK | PRIVACY | 쿠키·자동 수집, 트래킹 |

3. **현재 문서와 대조**
   - `docs/PRIVACY_{ja,kr}.md`, `docs/TERMS_{ja,kr}.md` 읽기
   - 변경된 기능이 이미 명시되어 있는지 grep
   - 누락/불일치 항목을 목록화

4. **수정 제안 출력**
   각 항목별로:
   - 근거 커밋 SHA + 파일 경로
   - 영향 문서 + 섹션 (예: `PRIVACY_ja.md §3 수집항목`)
   - 기존 문구 (있으면)
   - 제안 문구 (한국어 + 일본어 양쪽, reverb-jp 톤 유지)
   - 심각도: `must` (법적 필수) / `should` (권장) / `nice` (선택)

5. **승인 절차**
   - `AskUserQuestion`으로 항목별 적용 여부 묻기 (한 번에 여러 항목이면 multiSelect)
   - 승인된 항목만 `Edit` 도구로 문서 패치
   - 한/일 양쪽 동기화 (한 쪽만 수정 금지)

6. **완료 시 안내**
   - 수정된 파일 목록 + diff 요약
   - "이 변경은 운영 배포 공지/푸시 알림이 필요할 수 있음" 플래그 (중대 변경 시)
   - 커밋은 사용자가 명시적으로 요청할 때만 진행 (`docs: update privacy policy for X`)

## 제외 항목

- 미구현 기능은 반영하지 않는다 (구현되어 배포된 변경만 대상)
- 단순 오탈자/리팩토링/UI 문구 변경은 건너뛴다
- 이미 약관에 명시된 기능의 세부 구현 변경은 건너뛴다

## 주의

- PIPA(한국) + APPI(일본) 이중 준수 관점에서 검토
- 국외이전(일본→한국 Supabase)은 반드시 명시
- 일본 스테스 마케팅 규제(#PR, #広告) 관련 변경은 TERMS 반영 필수
- 약관 개정 시 "시행일" 명시 (문서 상단 갱신일 업데이트)
- 사용자에게 이미 받은 동의 범위를 초과하는 수집/이전이 추가되면 **재동의 절차**도 함께 제안
