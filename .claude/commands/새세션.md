# 새세션

REVERB JP에서 새 작업을 시작할 때 「전시장 펼치기 + 그 안으로 입장」을 한 번에 해주는 단축키.

## 무엇이 바뀌나 (중요)

- **기본 동작**: 전시장(별도 작업 폴더)을 만든 뒤, **지금 이 세션이 그 폴더로 바로 들어갑니다.** 새 터미널을 열 필요가 없습니다.
- 그래서 **작업 A를 끝낸 뒤(`/세션종료`로 마무리), 같은 터미널에서 `/새세션 B`** 를 부르면 곧장 다음 작업 전시장으로 넘어갈 수 있습니다.
- **동시에 두 작업을 병행**하려면(예: A를 멈추지 않고 B도 같이) — 그건 새 터미널이 필요합니다. 한 세션은 한 시점에 한 폴더에만 있을 수 있기 때문. 이때는 새 터미널을 열어 그 폴더에서 시작하세요.

## 비유

집을 짓는 중에 인부 여러 명이 같은 거실(메인 dev 브랜치)에서 동시에 작업하면 페인트와 벽지가 섞입니다.
이 명령어는 **본 거실 옆에 임시 전시장을 펼치고, 그 전시장으로 곧바로 들어가** 다른 작업과 충돌 없이 자기 공간에서 작업하게 해줍니다.

## 입력

`/새세션 <작업이름>`

- 작업이름은 한글·영문·하이픈 모두 가능. 공백은 자동으로 하이픈으로 변환.
- 예: `/새세션 공지개선`, `/새세션 campaign-fix`, `/새세션 결과물 엑셀 컬럼` (공백 → 하이픈)

## 메인 Claude 가 수행할 절차

### Step 1 — 입력 검증
- `$ARGUMENTS` 가 비어 있으면 작업 이름 묻기 (`AskUserQuestion`). 종료.
- 작업이름의 공백을 하이픈으로 치환하여 `SLUG` 변수에 저장.
  - 예: "결과물 엑셀 컬럼" → "결과물-엑셀-컬럼"
- `WORKTREE_PATH = ~/Documents/projects/reverb-jp-{SLUG}` (절대경로로 다룬다 — 전시장 안에서 실행될 때 상대경로 `../` 가 어긋나는 것 방지)
- `BRANCH_NAME = feature/{SLUG}`

### Step 2 — 현재 위치·충돌 검사
1. 현재 폴더가 메인 worktree인지, 이미 다른 전시장 안인지 확인:
   - 메인 worktree 경로: `git worktree list --porcelain | awk '/^worktree/{print $2; exit}'`
   - 현재 폴더: `pwd`
   - **두 값이 다르면 (= 이미 다른 전시장 안)**: 종료하지 말고 사용자에게 권고만 한다 —
     "현재 다른 전시장(`{현재경로}`) 안에 있습니다. 그 작업을 `/세션종료`로 먼저 마무리하고 메인 폴더로 돌아온 뒤 새 전시장을 펼치는 게 깔끔합니다(전시장 속 전시장은 정리가 꼬일 수 있음). 그래도 지금 바로 새 전시장으로 넘어갈까요?" → `AskUserQuestion`(지금 넘어가기 / 먼저 세션종료). "먼저 세션종료" 선택 시 종료.
2. 미커밋 변경 경고 (정보용, 차단 아님):
   - `git status --porcelain` 결과가 비어 있지 않으면: "현재 폴더에 저장 안 된 변경이 있습니다. 이 변경은 이 폴더(`{현재경로}`)에 그대로 남습니다(전시장 진입이 파일을 옮기지 않음). 새 전시장은 깨끗한 origin/dev 기반으로 시작됩니다." 한 줄 안내.
3. 같은 이름의 worktree가 이미 있는지: `git worktree list --porcelain | grep "reverb-jp-{SLUG}$"`
   - 있으면 안내 후 종료: "전시장 「{SLUG}」가 이미 펼쳐져 있습니다. (그 전시장으로 들어가려면 `EnterWorktree`의 path 진입을 직접 요청하세요.)"
4. 같은 이름의 브랜치가 이미 있는지: `git branch -a | grep "feature/{SLUG}$"`
   - 있으면 안내 후 종료: "브랜치 `feature/{SLUG}`가 이미 존재합니다. 다른 이름으로 만드세요."

### Step 3 — 전시장 펼치기 (git worktree add, 기존 방식 유지)
1. 최신 dev 가져오기:
   ```bash
   git fetch origin dev
   ```
2. worktree + feature 브랜치 동시 생성 (절대경로 + origin/dev 기반):
   ```bash
   git worktree add ~/Documents/projects/reverb-jp-{SLUG} -b feature/{SLUG} origin/dev
   ```
   - **EnterWorktree 의 `name` 으로 만들지 않는 이유**: `name` 방식은 `.claude/worktrees/` 안에 만들고 기본 분기 기준(baseRef)이 default 브랜치(main)일 수 있다. REVERB 관습은 `reverb-jp-{SLUG}` 폴더 + **origin/dev 기반**이므로, 폴더·기준 브랜치를 그대로 유지하려고 `git worktree add` 로 직접 만든 뒤 아래 `path` 로 진입한다.
3. 결과 확인:
   ```bash
   git worktree list
   ```

### Step 4 — 현재 세션을 그 전시장으로 진입
- `EnterWorktree` 도구를 `path` 인자로 호출:
  - `path = ~/Documents/projects/reverb-jp-{SLUG}` (Step 3에서 만든 절대경로)
- 이러면 새 터미널 없이 **지금 세션의 작업 폴더가 그 전시장으로 전환**된다.
- (참고: `path` 진입으로 들어간 전시장은 `ExitWorktree` 가 자동 삭제하지 않는다. 정리는 작업 종료 후 `/세션종료` → PR → 메인에서 수동 remove 흐름을 따른다.)

### Step 5 — 사용자 안내 (한국어, 이모지 사용 안 함)

```
[전시장 입장 완료] 「{SLUG}」

작업 폴더: ~/Documents/projects/reverb-jp-{SLUG}
브랜치   : feature/{SLUG} (origin/dev 기반)

이제 이 세션이 위 전시장 안에 있습니다. 바로 작업을 시작하세요.
 - 코드 수정 → 빌드(cd dev && bash build.sh) → 커밋 → push 모두 이 전시장 안에서.
 - 작업이 끝나면 이 세션에서 /세션종료 호출 → PR 생성 + 메인 폴더 복귀.

[참고] 동시에 다른 작업도 병행하려면, 그건 새 터미널을 열어 시작하세요.
       (한 세션은 한 폴더에만 있을 수 있어 동시 병행은 세션이 하나 더 필요합니다.)
```

> 만약 이 세션이 **고문 세션**(메인 폴더에서 규칙·검증 담당)이었다면, 진입 후에는 개발 세션으로 바뀐다는 점을 한 줄 덧붙인다.

## 끝낸 뒤

작업이 끝나면 이 세션에서 `/세션종료` 호출 → PR 생성 + (worktree에서 나와 메인 폴더로 복귀). PR 머지 후 메인 폴더에서 정리:

```bash
cd ~/Documents/projects/reverb-jp
git worktree remove ~/Documents/projects/reverb-jp-{SLUG}
git branch -D feature/{SLUG}
git pull origin dev
```

그 다음 같은 터미널에서 `/새세션 다음작업` 으로 바로 이어갈 수 있다.

## 실패·예외 처리

- `git fetch` 실패 시: 네트워크/원격 권한 확인 안내
- `git worktree add` 실패 시: 에러 메시지 그대로 전달 + 메인 폴더가 git 저장소인지 확인 안내
- `EnterWorktree` 진입 실패 시: 만들어진 전시장 경로를 안내하고, 새 터미널로 그 폴더에서 시작하는 기존 방법으로 폴백
- `gh` 미설치/미인증: `/세션종료` 단계에서 명시. `/새세션` 단계에는 영향 없음

## 사용 시점

- ✅ 새 작업을 시작할 때 — 메인 폴더든 직전 작업을 끝낸 뒤든, 이 한 줄로 전시장 펼치고 바로 입장
- ✅ 작업 A 종료 후 같은 터미널에서 작업 B로 넘어갈 때 (순차)
- ✅ 동시에 두 작업 병행 — 이때만 새 터미널을 추가로 사용 (전시장은 `/새세션`으로 만들어두고 새 터미널에서 그 폴더 열기)
- ❌ 한 번에 한 작업만 하고 전시장 격리가 굳이 필요 없다면 — 메인 폴더에서 그대로 작업해도 됨
