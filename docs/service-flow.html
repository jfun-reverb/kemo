3캠페인 상태 라이프사이클

캠페인의 6가지 상태(draft→scheduled→active→closed→ended/expired)와 전환 조건, 노출 규칙
stateDiagram-v2
  [*] --> draft: 캠페인 등록

  draft --> scheduled: 관리자 상태 변경
  draft --> active: 관리자 상태 변경 (deadline 유효시)

  scheduled --> active: 관리자 수동 변경 또는 recruit_start 도래 (자동)
  scheduled --> draft: 관리자 상태 변경

  active --> closed: 관리자 수동 종료
  active --> closed: deadline 경과 (자동)

  closed --> ended: submission_end 경과 (자동, autoEndCampaigns)
  closed --> expired: 운영자 노출 토글 OFF (수동)
  expired --> closed: 운영자 노출 토글 ON (자연 상태 재계산)
  ended --> expired: 운영자 노출 토글 OFF (수동)
  closed --> draft: 관리자 재등록

  note right of draft
    준비중 — 비공개
    목록 미노출
  end note
  note right of scheduled
    공개예정 — 목록 노출
    신청 불가
    recruit_start 도래 시 active 자동 전환
    (autoOpenCampaigns)
  end note
  note right of active
    모집중 — 목록 노출
    신청 가능
    deadline 경과 시 자동 closed
  end note
  note right of closed
    모집마감 — 募集締切 오버레이
    목록 노출 유지 (운영자 토글 OFF 전까지)
    인플루언서 화면: 신청 불가, 상세 조회 가능
    결과물 제출 기간 진행 중
    submission_end 경과 시 ended 자동 전환
  end note
  note right of ended
    종료 — 終了 오버레이(검은색)
    목록 노출 유지 (운영자 토글 OFF 전까지)
    결과물 제출 마감됨
    자동 전이 없음 (수동 토글로만 expired)
  end note
  note right of expired
    비노출 — 운영자 수동 전환만
    자동 전이 없음
  end note
2026-04-27 (migration 072): campaigns.recruit_start date NULL 신규 컬럼. autoOpenCampaigns() → autoCloseCampaigns() 순차 실행 — scheduled 캠페인의 recruit_start ≤ now() 이면 active 자동 전환. recruit_start IS NULL 이면 인플루언서 화면에서 "오늘 ~ deadline" 폴백 렌더.
2026-05-18 (migration 129): post_deadline 컬럼 삭제 + autoExpireCampaigns 제거. 캠페인 노출 관리는 운영자 수동 토글 — ON 시 closed 자연 상태 재계산, OFF 시 status=expired. expired 진입은 운영자 토글 OFF 만, 자동 전이 없음.
2026-05-27 (migration 156): ended 상태 분리. submission_end 경과 시 closed → ended 자동 전환 (autoEndCampaigns()). 인플루언서 화면에 終了 오버레이(검은색) 노출. closed 는 募集締切(핑크) 오버레이 유지.
