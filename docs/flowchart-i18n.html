REVERB JP — i18n (다국어) 흐름도

최종 업데이트: 2026-04-16 | Phase 1 + Phase 2 통합

1. 언어 전환 흐름
사용자: 마이페이지 → 언어 메뉴 클릭
KO(한국어) 또는 JA(일본어) 선택
setLang('ko' 또는 'ja')
→ localStorage('reverb.lang') 저장
applyI18n() 실행
DOM 전체 data-i18n / data-i18n-html 스캔
현재 페이지의 모든 라벨이
선택 언어로 즉시 전환
JS 동적 영역(토스트, 모달)은
다음 렌더링 시 t(key)로 반영
2. t() 헬퍼 동작 흐름
코드에서 t('apply.loginRequired') 호출
getLang() → 현재 언어 확인 (ko/ja)
해당 언어 사전에서 키 검색
(ja.js 또는 ko.js)
키 존재 → 번역문 반환
"로그인해주세요"
키 없음 → 키 이름 그대로 반환
"apply.loginRequired"
(개발 중 누락 감지)
3. DB 에러 메시지 로케일 대응 흐름
Supabase 에러 발생
(예: 23505 unique_violation)
friendlyErrorJa(errorCode, getLang())
getLang() 결과
lang = 'ja'
"이미 등록되어 있습니다 (일본어)"
lang = 'ko'
"이미 등록되어 있습니다"
토스트(toast)로 사용자에게 표시

5. 파일 구조
dev/
├── lib/
│   └── i18n/
│       ├── index.js      ← 런타임 (getLang, setLang, t, applyI18n)
│       ├── ja.js          ← 일본어 사전 (~150 키)
│       └── ko.js          ← 한국어 사전 (~150 키)
├── js/
│   ├── app.js            ← gnb.*, home.*, notif.*
│   ├── auth.js           ← auth.*
│   ├── campaign.js       ← campaign.*, detail.*
│   ├── application.js    ← apply.*, activity.*, delivStatus.*
│   ├── mypage.js         ← mypage.*, activity 일부
│   └── ui.js             ← friendlyErrorJa() (에러 로케일)
└── index.html            ← data-i18n / data-i18n-html 속성
