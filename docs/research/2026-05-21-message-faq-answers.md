## 작성 원칙 (사양서 §7 — 반드시 준수)

> **인플루언서는 시스템·개발 이해도가 매우 낮다고 전제한다. 초등학생에게 설명하듯 쉽게, 처리 방향은 번호 단계(1·2·3)로 끊어서 안내한다.** (2026-05-21 사용자 명시)

- **쉬운 말로:** 전문용어·시스템 용어·영어 약어 금지. 부득이하면 동작으로 풀어 설명("인증" → 「メールのリンクを開く」)
- **한 문장 = 한 동작:** 짧게 끊는다. 한 문장에 할 일 2개 넣지 않기
- **할 일이 있으면 번호 단계:** "무엇을 → 어디서 → 어떻게"를 `1. / 2. / 3.`로. 누르는 버튼은 「」로 정확히
- **마지막은 안전망:** 막혔을 때 「下の「直接お問い合わせ」からご連絡ください」로 닫기(handoff 질문 제외)
- **한국어·일본어는 의미 동일·단계 수 동일** (한쪽만 단계가 더 많으면 안 됨)

각 질문은 아래 6칸으로 구성됩니다.

| 칸 | 의미 |
|---|---|
| **質問 (일본어 제목)** | 인플루언서가 선택지 트리에서 보는 질문 제목 |
| **回答 (일본어 답변)** | 인플루언서에게 실제로 보내는 답변 본문 (단계별·쉬운 톤) |
| **질문 (한국어)** | 관리자가 등록·관리할 때 이해용 |
| **답변 (한국어)** | 일본어 답변의 의미. 단계 수·순서 동일 |
| **화면 이동** | 답변 카드에 붙이는 앱 화면 바로가기 버튼 (없으면 「-」) |
| **사람 연결 / 단계** | `handoff`(바로 직접 문의) 여부 + 맞춤 노출 단계 태그 |

> **표기 주의:** 일본어 답변 안의 「マイページ」「応募履歴」 등 화면 이름은 **앱 UI 라벨과 정확히 일치**해야 합니다. UI 라벨이 바뀌면 이 문안도 함께 수정합니다.
> **단계 태그 enum (사양서 §3-3):** `pending` / `approved_purchase` / `approved_visit` / `approved_post` / `rejected` / `done` (6종). NULL = 모든 단계.

## 카테고리 1. 申し込み・応募 (신청·응모)

### Q1-1. 応募ができません (신청이 안 돼요)

> 이 질문은 원인이 5가지라 **하위 선택지**로 나눕니다 (트리 2단). 아래 5개가 Q1-1의 자식 항목입니다.

#### Q1-1-a. メール認証が必要と表示されます (이메일 인증이 필요하다고 떠요)

- **回答 (일본어):**
  「ご応募の前に、メールアドレスの確認が必要です。つぎの順番でおこないます。
  1. ご登録のメールアドレスに届いている、REVERBからのメールを開きます。
  2. メールの中のリンクを1回押します。
  3. これで確認は完了です。もう一度ご応募をお試しください。
  メールが見つからないときは、迷惑メールのフォルダもご確認ください。それでも届いていないときは、下の「直接お問い合わせ」からご連絡ください。」
- **질문 (한국어):** 이메일 인증이 필요하다고 떠요
- **답변 (한국어):** 응모 전에 이메일 주소 확인이 필요합니다. 1. 가입한 이메일로 온 REVERB 메일을 엽니다. 2. 메일 안의 링크를 한 번 누릅니다. 3. 확인 완료, 다시 응모해 보세요. 메일이 안 보이면 스팸함도 확인하시고, 그래도 없으면 직접 문의 바랍니다.
- **화면 이동:** -
- **사람 연결 / 단계:** 메일 미수신 시 handoff 안내 / 단계 무관

#### Q1-1-b. 必須情報を入力してくださいと表示されます (필수 정보를 입력하라고 떠요)

- **回答 (일본어):**
  「ご応募の前に、マイページで3つの情報を登録しておく必要があります。下のボタンからマイページを開いて、つぎを登録してください。
  1. SNSアカウント（応募するキャンペーンの対象になっているもの）
  2. お届け先（郵便番号・都道府県・市区町村・電話番号）
  3. PayPalのメールアドレス
  まだ登録していない項目には「未登録」と表示されます。すべて登録すると、ご応募いただけます。」
- **질문 (한국어):** 필수 정보를 입력하라고 떠요
- **답변 (한국어):** 응모 전에 마이페이지에서 3가지를 등록해야 합니다. 아래 버튼으로 마이페이지를 열어 1. SNS 계정(응모할 캠페인 대상) 2. 배송지(우편번호·도도부현·시·전화) 3. PayPal 이메일을 등록하세요. 미등록 항목은 「未登録」으로 표시되며, 모두 등록하면 응모할 수 있습니다.
- **화면 이동:** 基本情報 (기본정보)
- **사람 연결 / 단계:** - / 단계 무관

#### Q1-1-c. フォロワー数が足りないと表示されます (팔로워 수가 부족하다고 떠요)

- **回答 (일본어):**
  「このキャンペーンには「最低フォロワー数」の条件があります。
  ・必要なフォロワー数：{required}人以上
  ・あなたの現在のフォロワー数：{current}人
  つぎをご確認ください。
  1. 下のボタンからマイページのSNSアカウントを開きます。
  2. フォロワー数を、いまの最新の人数に直します。
  3. 直したあと、もう一度ご応募をお試しください。
  条件は、代表に設定したSNSのフォロワー数で確認します。」
- **질문 (한국어):** 팔로워 수가 부족하다고 떠요
- **답변 (한국어):** 이 캠페인은 최소 팔로워 수 조건이 있습니다. ·필요 팔로워: {required}명 이상 ·현재 팔로워: {current}명. 1. 아래 버튼으로 마이페이지 SNS 계정을 엽니다. 2. 팔로워 수를 지금 최신 인원으로 고칩니다. 3. 고친 뒤 다시 응모해 보세요. 조건은 대표로 설정한 SNS의 팔로워 수로 확인합니다.
- **화면 이동:** SNSアカウント (SNS 등록)
- **사람 연결 / 단계:** - / 단계 무관
- **⚙️ 동적 치환:** `{required}` = 캠페인 `min_followers`, `{current}` = 본인 대표 SNS 팔로워 수. **응모 차단 시점**(응모 버튼이 막혔을 때)에 이 답변을 띄우면 본인이 얼마나 모자란지 즉시 안다. 두 값 모두 클라이언트에 이미 있어 신규 서버 호출 불필요. 값이 없으면(데이터 누락) 두 줄을 생략하고 일반 안내만 노출.

#### Q1-1-d. すでに応募済みと表示されます (이미 신청했다고 떠요)

- **回答 (일본어):**
  「同じキャンペーンには、重ねてご応募いただけません。すでにご応募されている可能性があります。下のボタンから応募履歴を開くと、いまの応募の状況をご確認いただけます。」
- **질문 (한국어):** 이미 신청했다고 떠요
- **답변 (한국어):** 같은 캠페인에 중복 응모는 안 됩니다. 이미 응모하셨을 수 있습니다. 아래 버튼으로 응모이력을 열면 현재 응모 상태를 확인할 수 있습니다.
- **화면 이동:** 応募履歴 (응모이력)
- **사람 연결 / 단계:** - / 단계 무관

#### Q1-1-e. 募集が締め切られたと表示されます (모집이 마감됐다고 떠요)

- **回答 (일본어):**
  「このキャンペーンは、募集が終わっています。募集人数がいっぱいになったか、募集の期限が過ぎた場合は、新しくご応募いただけません。ほかのキャンペーンは、キャンペーン一覧からご覧いただけます。」
- **질문 (한국어):** 모집이 마감됐다고 떠요
- **답변 (한국어):** 이 캠페인은 모집이 끝났습니다. 모집 인원이 찼거나 마감일이 지나면 새로 응모할 수 없습니다. 다른 캠페인은 캠페인 목록에서 볼 수 있습니다.
- **화면 이동:** -
- **사람 연결 / 단계:** - / 단계 무관

### Q1-2. 応募を取り消したいです (응모를 취소하고 싶어요)

- **回答 (일본어):**
  「ご応募の取り消しは、つぎの順番でおこないます。
  1. 下のボタンから応募履歴を開きます。
  2. 取り消したい応募のカードにある、メニュー（点が3つのマーク）を押します。
  3. 「取り消し」を選びます。
  なお、成果物がすでに承認されている場合などは、取り消しができないことがあります。とちゅうでエラーが出るときは、下の「直接お問い合わせ」からご連絡ください。」
- **질문 (한국어):** 응모를 취소하고 싶어요
- **답변 (한국어):** 응모 취소는 다음 순서로 합니다. 1. 아래 버튼으로 응모이력을 엽니다. 2. 취소할 응모 카드의 메뉴(점 3개 표시)를 누릅니다. 3. 「취소」를 고릅니다. 단 결과물이 이미 승인된 경우 등은 취소가 안 될 수 있습니다. 도중에 에러가 나면 직접 문의 바랍니다.
- **화면 이동:** 応募履歴 (응모이력)
- **사람 연결 / 단계:** 에러 시 handoff / 단계 무관
- **⚠️ 운영 메모:** 실측에서 "취소가 에러로 안 됨" 사례 있었음. **취소 버그 수정 완료 전까지** 이 답변은 에러 시 handoff를 강조한 현재 문안 유지. 수정 후 안내 정확화.

### Q1-3. もう一度応募できますか (다시 신청할 수 있나요)

- **回答 (일본어):**
  「ご応募を取り消したあと、そのキャンペーンの募集がまだ続いていれば、もう一度ご応募いただけます。募集がすでに終わっている場合は、新しくご応募いただけません。」
- **질문 (한국어):** 취소 후 다시 신청할 수 있나요
- **답변 (한국어):** 응모를 취소한 뒤, 그 캠페인 모집이 아직 진행 중이면 다시 응모할 수 있습니다. 모집이 이미 끝난 경우엔 새로 응모할 수 없습니다.
- **화면 이동:** -
- **사람 연결 / 단계:** - / 단계 무관

## 카테고리 2. 審査・結果 (심사·결과)

### Q2-1. 審査はいつ終わりますか (심사는 언제 끝나나요)

- **回答 (일본어):**
  「いまの審査の状況は、この画面のいちばん上、または応募履歴でご確認いただけます。審査には数日かかることがあります。結果が決まりましたら、アプリの通知でお知らせしますので、今しばらくお待ちください。」
- **질문 (한국어):** 심사는 언제 끝나나요
- **답변 (한국어):** 현재 심사 상태는 이 화면 맨 위, 또는 응모이력에서 확인할 수 있습니다. 심사에 며칠 걸릴 수 있고, 결과가 정해지면 앱 알림으로 안내하니 잠시 기다려 주세요.
- **화면 이동:** 応募履歴 (응모이력)
- **사람 연결 / 단계:** - / **단계: pending(심사중)**

### Q2-2. 当選したかどうか、どこで分かりますか (당첨됐는지 어떻게 아나요)

- **回答 (일본어):**
  「当選の結果は、つぎの場所でご確認いただけます。
  1. 下のボタンから応募履歴を開きます。
  2. 各応募のところに表示される状態を見ます。
  当選された場合は「当選」と表示され、アプリの通知でもお知らせします。」
- **질문 (한국어):** 당첨됐는지 어디서 알 수 있나요
- **답변 (한국어):** 당첨 결과는 다음에서 확인합니다. 1. 아래 버튼으로 응모이력을 엽니다. 2. 각 응모에 표시되는 상태를 봅니다. 당첨 시 「当選」으로 표시되고 앱 알림으로도 안내합니다.
- **화면 이동:** 応募履歴 (응모이력)
- **사람 연결 / 단계:** - / **단계: pending**

### Q2-3. 選ばれなかった理由を知りたいです (왜 비승인(선정 안 됨)됐는지 알고 싶어요)

- **回答 (일본어):**
  「ご応募ありがとうございました。選考は、フォロワー数だけでなく、投稿の内容やキャンペーンとの相性など、いくつかの点をまとめて判断しております。そのため、おひとりずつの個別の理由はお伝えしておりません。ご縁がありませんでしたが、ぜひまた別のキャンペーンにご応募ください。お待ちしております。」
- **질문 (한국어):** 왜 비승인(선정 안 됨)됐는지 알고 싶어요
- **답변 (한국어):** 응모 감사합니다. 선정은 팔로워 수만이 아니라 게시물 내용·캠페인과의 적합도 등 여러 점을 종합 판단합니다. 그래서 개별 사유는 안내드리지 않습니다. 이번엔 인연이 없었지만, 꼭 다른 캠페인에 또 응모해 주세요. 기다리겠습니다.
- **화면 이동:** -
- **사람 연결 / 단계:** - / **단계: rejected(비승인)**
- **⚠️ 정책 메모 (2026-05-21 결정):** 응모 비승인 사유는 ①저장 칸이 없고 ②종합·정성 판단이라 자동 산출 불가하며 ③비선정자 개별 미통보가 현 정책(캠페인 안내 `notice6`). 따라서 **구체 사유는 안내하지 않고 완곡 답변만** 한다. "모집조건 미달 안내"는 비승인 시점이 아니라 **응모 차단 시점(Q1-1-c)**에서 구체 수치로 해소한다 (응모가 됐다는 건 이미 정량 조건을 통과했다는 뜻이므로).

## 카테고리 3. 成果物の提出 (결과물 제출)

### Q3-1. レシート(領収書)はどう提出しますか (영수증을 어떻게 올리나요)

- **回答 (일본어):**
  「レシートのご提出は、つぎの順番でおこないます。
  1. 商品を購入します。
  2. 下の「成果物提出」ボタンを押して、提出の画面を開きます。
  3. つぎの3つを入力します。
  　・注文番号
  　・購入した日
  　・購入した金額
  4. レシートの写真をアップロードします。
  5. 「提出」を押したら完了です。
  わからないときは、下の「直接お問い合わせ」からご連絡ください。」
- **질문 (한국어):** 영수증을 어떻게 올리나요
- **답변 (한국어):** 영수증 제출은 다음 순서로 합니다. 1. 상품을 구매합니다. 2. 아래 「成果物提出」 버튼을 눌러 제출 화면을 엽니다. 3. 주문번호·구매한 날짜·구매한 금액 3가지를 입력합니다. 4. 영수증 사진을 올립니다. 5. 「제출」을 누르면 완료. 모르겠으면 직접 문의 바랍니다.
