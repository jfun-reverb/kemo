# REVERB JP 업데이트 보고서

> 작성일: 2026년 4월 8일

---

## 요약

인플루언서 앱과 관리자 페이지 전반에 걸쳐 사용성 개선 및 신규 기능을 추가했습니다.
주요 변경 사항은 마이페이지 구조 개편, 관리자 대시보드 강화, 캠페인 관리 기능 확장입니다.

---

## 1. 인플루언서 앱 (모바일)

### 1.1 상단 메뉴 간소화

- **변경 전**: 로그인 후 상단에 사용자 이름, 마이페이지, 로그아웃 버튼이 모두 표시
- **변경 후**: 상단 메뉴를 깔끔하게 정리. 마이페이지와 로그아웃은 하단 탭의 My Page에서 접근
- 관리자 계정으로 로그인 시에만 "Admin" 버튼 표시

### 1.2 마이페이지 구조 변경

- **변경 전**: 가로 탭 방식 (応募履歴 | 基本情報 | SNS | 配送先 | 振込口座)
- **변경 후**: 메뉴 리스트 → 상세 페이지 방식
  - 메뉴 항목: 応募履歴, 基本情報, SNSアカウント, 配送先, 振込口座, パスワード変更, ログアウト
  - 각 메뉴를 선택하면 해당 상세 페이지로 이동, ← 뒤로가기로 복귀
  - 退会(탈퇴) 버튼은 하단에 작게 배치

### 1.3 캠페인 조회수 추가

- 인플루언서가 캠페인 상세를 열 때마다 조회수가 자동으로 1 증가
- 관리자 페이지에서 캠페인별 조회수 확인 가능

### 1.4 모집 마감 캠페인 노출

- 모집 기간은 끝났지만 게시(SNS 포스팅) 기한이 남은 캠페인은 목록에 계속 노출
- "募集締切(모집 마감)" 라벨로 표시, 신청 버튼은 비활성화

## 2. 관리자 페이지 (PC)

### 2.1 대시보드 강화

#### 회원가입 추이 차트

- 일별 신규 가입자 수를 막대 차트로 시각화
- 기간 선택: 7일 / 30일 / 전체 (기본값 30일)

#### 신규 가입 현황

- 오늘 가입자 수, 이번주 가입자 수를 한눈에 확인
- 이번주 기간 표시 (예: 4/1 ~ 4/8)

#### 프로필 완성률

- 전체 인플루언서의 정보 입력 현황을 백분율로 표시
- SNS 전체 및 채널별 (Instagram, X, TikTok, YouTube)
- 배송지 등록률, 계좌 등록률

### 2.2 캠페인 목록 개선

#### 썸네일 이미지

- 캠페인 목록에 대표 이미지가 작은 사진으로 표시
- 등록된 이미지 수가 좌측 하단에 "+3" 형태로 표시

#### 필터 및 검색

- **상태 필터**: 전체 / 준비 / 모집예정 / 모집중 / 일시정지 / 종료
- **타입 필터**: 전체 / 리뷰어 / 기프팅
- **검색**: 캠페인명 또는 브랜드명으로 실시간 검색

#### 정렬

- 조회수, 신청수, 등록일, 수정일 기준 오름차순/내림차순 정렬
- 각 항목 헤더 옆 ▲▼ 버튼으로 전환

#### 순서 변경 모드

- "순서 변경" 버튼 클릭 시 모든 필터/정렬이 초기화되고 ↑↓ 화살표 활성화
- 순서 변경 시 즉시 반영 (이동된 행이 잠깐 강조 표시)
- "순서 변경 완료" 버튼으로 모드 종료

#### 상태 변경

- 기존: 상태 클릭 시 순서대로 순환 변경
- 변경: 상태 클릭 시 드롭다운 목록에서 원하는 상태를 직접 선택

#### 날짜 및 D-day 표시

- 등록일, 수정일(시:분까지) 컬럼 추가
- 캠페인명 아래에 게시 마감일, 신청 영역에 모집 마감일 표시
- 각 날짜 옆에 D-day 라벨 (D-7, D-Day, D+3 등)
  - 초과: 빨간색 / 3일 이내: 노란색 / 여유: 회색

### 2.3 캠페인 자동 상태 관리

#### 자동 종료

- 모집 마감일이 지난 캠페인은 자동으로 "종료" 상태로 변경
- 페이지 로드 시 자동 체크

#### 마감일 경과 보호

- 모집 마감일이 지난 캠페인은 "모집중" 상태로 되돌릴 수 없음
- 편집 저장, 상태 드롭다운 변경 시 모두 차단

#### 마감일 입력 검증

- 게시 마감일이 모집 마감일보다 빠르면 경고 문구가 입력창 아래에 즉시 표시
- 날짜를 올바르게 수정하면 경고 자동 소멸
- 잘못된 상태로는 저장 불가

### 2.4 사이드바 개선

- 메뉴 아이콘을 이모지에서 Material Design 아이콘으로 교체
- 햄버거 버튼(☰)으로 사이드바 접기/펼치기 가능
- 접힌 상태에서는 아이콘만 표시되어 작업 영역이 넓어짐

### 2.5 버튼 반응 개선

- 모든 버튼에 마우스 올림(hover), 클릭(active) 시 시각적 피드백 추가
- 순서 변경 시 이동된 행이 잠깐 핑크색으로 강조

### 2.6 새로고침 깜빡임 해결

- 관리자 페이지에서 새로고침 시 대시보드가 잠깐 보였다가 원래 페이지로 전환되던 문제 수정
- 이제 새로고침해도 보던 페이지가 바로 표시

### A. 캠페인 성과물 제출 구조 (🔥 최우선)

**배경**

- 현재는 리뷰어(モニター)형 캠페인에서 영수증만 등록 가능
- 기프팅(ギフティング)형은 **SNS 게시물 URL** 제출이 필요하나 현재 화면/스키마 미구현
- 향후에는 영수증 + SNS URL 모두 요구하는 혼합형 캠페인이 등장할 가능성

**결정해야 할 주제**

1. **캠페인별 필수 제출물 설정**

   - 캠페인 등록 폼에 "제출물 요구 항목" 체크박스 추가 여부
     - (a) 영수증 / (b) SNS 게시물 URL / (c) 둘 다 / (d) 기타(스크린샷 등)
   - 캠페인 단위로 제출물 종류를 유동적으로 지정할지, 캠페인 타입(리뷰어/기프팅)으로 고정할지
2. **인플루언서 활동관리 화면 폼 구조**

   - (A) **단일 통합 폼** — 영수증·URL·기타 항목을 한 화면에서 모두 입력
     - 장점: 화면 전환 없음, 진행 현황 한눈에
     - 단점: 사용자가 현재 캠페인에 필요없는 필드까지 보여 혼란 가능
   - (B) **단계별 분리 폼** — 영수증 단계 → SNS URL 단계 순차 제출
     - 장점: 한 번에 하나만 보여 명확, 상태 관리 단순
     - 단점: 단계가 많아지면 이탈률 증가 가능
   - (C) **캠페인 타입별 분기** — 리뷰어는 영수증 폼, 기프팅은 URL 폼, 혼합형은 통합 폼
     - 장점: 각 타입에 최적화
     - 단점: 화면 종류가 증가, 코드 분기 복잡
3. **SNS URL 검증 범위**

   - URL 형식 검증만 / 연결된 SNS 계정과 일치 여부까지 검증 / 스크린샷 추가 요구
4. **스키마 변경**

   - `receipts` 테이블 외에 `campaign_submissions` 같은 통합 테이블로 재설계할지
