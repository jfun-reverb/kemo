## 배경
- 인플루언서 페이지 현재 일본어 하드코딩
- 개발서버에서 테스트하는 테스터들이 일본어를 몰라 QA 어려움
- 1차: 개발서버에 KO/JA 토글 도입 → 테스터 편의
- 2차: 운영서버에 배포하되 JA 기본, KO는 옵션

## 영향 파일
- `dev/index.html` — 하드코딩된 일본어 수백 개
- `dev/js/*.js` — alert/toast/렌더 텍스트
- 신규: `dev/lib/i18n/{ko,ja,index}.js`
- `dev/build.sh` — 새 i18n 파일 빌드 등록
- `CLAUDE.md`, `.claude/rules/ui.md` — 규칙 개정 필요

## i18n 인프라 설계
- **저장**: JS 객체 파일 (`window.I18N_KO = {...}`), CDN/바닐라에 최적
- **키**: `dot.case` 계층형 (`home.hero.title`, `common.save`)
- **적용**: 하이브리드
  - 정적 HTML: `data-i18n="..."` 속성 + `applyI18n()` 스캐너
  - 동적 JS: `t('...')` 헬퍼
- **폴백**: 키 없으면 JA → 없으면 키 자체 표시

## UI
- 위치: 마이페이지 메뉴 상단 (パスワード変更 위)
- 형태: 세그먼트 스위치 `[日本語] [한국어]`
- 라벨: `言語 / 언어` 병기

## 언어 결정 로직
```
1. localStorage('reverb.lang') 있으면 사용
2. 없으면 'ja' 기본
```
- navigator.language 자동 감지 사용 안 함 (운영 안전)
- dev 환경에서만 `?lang=ko` 쿼리 허용

### Phase 3 — 액션

## 운영 영향 최소화
- prod 기본값 `ja` 고정
- KO는 명시적 액션으로만 활성화
- 일반 JA 사용자 눈에 잘 안 띔

## 빌드/배포
- build.sh 수정 필요 (3개 JS 등록)
- 루트 index.html 용량 +30~50KB (두 언어 인라인)
- admin 영향 없음

## 지역 특이 처리
- 날짜: `formatDate()`에 lang 반영 (ja-JP / ko-KR)
- 우편번호/가나 이름/주소: 필드 형식은 JA 기준 유지, 라벨만 번역
- 금액(円), 전화번호 포맷: JA 기준 유지, 라벨만 번역

## 규칙 개정 필요
- CLAUDE.md: "인플루언서 페이지 UI 텍스트: 일본어" → "기본 일본어, KO 토글 지원"
- .claude/rules/ui.md: 동일
- lang="ja" 고정 규칙 → 동적 변경
- 날짜 포맷 ja-JP 고정 → lang 기반

## lookup_values
- 이미 `name_ko`, `name_ja` 존재 → `getLookupLabel(item)` 헬퍼로 현재 lang 반환
- `name_ko` 비면 `name_ja`로 폴백
