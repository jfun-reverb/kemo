# 결과물 관리 기획서 (Deliverables Management)

> **Status**: 기획 확정, DB 구조·UI 와이어프레임·구현 범위는 후속 세션에서 결정
> **Date**: 2026-04-15
> **Supersedes**: `docs/plan_receipt_management.md` (영수증 한정 초안)

기존 "영수증 관리"를 **"결과물(deliverables) 관리"** 로 확장. 캠페인 타입별로 제출 형태(영수증 / 게시물 URL)가 다르지만 검수 플로우는 공통으로 관리.

---

## 1. 용어 정리

| 용어 | 정의 |
|------|------|
| 결과물(deliverable) | 인플루언서가 캠페인 수행 후 제출하는 증빙 (영수증 or 게시물 URL) |
| 심사(application review) | 신청자를 선정/비선정 결정 |
| 검수(deliverable review) | 제출된 결과물 승인/반려 결정 |
| OT 시트 | 기프팅/방문형 선정자에게 보내는 오리엔테이션 안내 (이번 범위 제외, 발송 체크만) |

---

## 2. 타입별 플로우

| 단계 | 리뷰어(monitor) | 기프팅(gifting) / 방문형(visit) |
|------|---|---|
| 신청 | 인플루언서 신청 | 인플루언서 신청 |
| 심사 | 관리자 승인/비승인 | 관리자 승인/비승인 |
| 선정 이후 | (캠페인 정보만 노출) | OT 시트 이메일 전달 (수동, 외부) |
| 실행 | 제품 구매 | 기프팅: 배송 체험 / 방문형: 매장 방문 |
| 제출 | 영수증 이미지 + 구매일 + 금액 | SNS 게시물 URL |
| 검수 | 영수증 대조 → 승인/반려 | 게시물 확인 → 승인/반려 |
| 완료 | 승인 = 완료 (정산 이후는 범위 외) | 승인 = 완료 |

## 3. 관리자 3개 작업 모드

역할을 분리해 중복을 피한다.

| 화면 | 위치 | 기본 필터 | 주 용도 |
|------|------|-----------|---------|
| 신청관리 페인 | 사이드바 | `status=pending` | 전 캠페인 심사대기 몰아처리 |
| 결과물 관리 페인 (신설) | 사이드바 | `status=pending` | 전 캠페인 검수대기 몰아처리 |
| 캠페인 진행현황 탭 (신설) | 캠페인 상세 내 | 해당 캠페인 전체 | 캠페인 단위 전체 흐름 파악 |

### 3-1. 신청관리 페인 (기존 역할 축소)
- 기본 필터 `pending` 고정, 전체 보기는 진행현황 탭으로 유도
- 기능: 승인/비승인 + 되돌리기 (기존 유지)

### 3-2. 결과물 관리 페인 (신설)
- 리스트: 제출일·캠페인·인플루언서·타입 아이콘(영수증/URL)·상태·썸네일
- 필터: 캠페인, 타입(receipt/post), 상태, 인플루언서 검색
- 정렬 기본: **제출일 오래된 순** (방치 방지)
- 상세 모달:
  - 타입별 UI 분기 (영수증=이미지+금액 / URL=링크·새창 열기·추후 embed)
  - 이전 제출 이력 타임라인 표시
  - 승인 / 반려(사유 입력) 액션

### 3-3. 캠페인 진행현황 탭 (신설)
- 캠페인 상세 페이지 내 신규 탭 (편집·미리보기와 병렬)
- 열: 인플루언서 / 신청상태 / OT체크박스(기프팅·방문형만) / 결과물 상태 / 액션
- 한 화면에서 심사·OT체크·검수 전부 가능
- OT 토글: 체크는 1클릭, 해제는 확인 모달

## 4. 인플루언서 UX

### 4-1. 결과물 제출 폼 (활동관리)
recruit_type 분기:
- **monitor**: 영수증 이미지 + 구매일 + 금액
- **gifting / visit**: SNS 게시물 URL (복수 제출 가능)
  - URL 자동판별: `instagram.com`/`tiktok.com`/`youtube.com`·`youtu.be`/`x.com`·`twitter.com`/`qoo10.jp`
  - 판별 실패: 드롭다운으로 채널 수동 선택

### 4-2. 반려 사유 노출 (3중 알림, 역할 분리)
| 위치 | 역할 | 소멸 조건 |
|------|------|-----------|
| 마이페이지 상단 알림 섹션 | 일회성 공지 | 클릭 이동 시 / 재제출 pending 복귀 시 |
| 응모이력 리스트 "반려" 배지 | 상태 지속 표시 | 상태 바뀌면 자동 |
| 활동관리 카드 빨간 배너 | 액션 유도 (수정 재제출) | 재제출 pending 복귀 시 |

### 4-3. 재제출
- pending 중: 자유롭게 덮어쓰기
- 반려 후: 재제출 → 다시 pending
- 승인 후: 관리자 되돌리기 필요 → 그 후 pending 복귀

## 5. 상태 체계 & 재제출 정책

### 5-1. 상태
- `applications.status`: `pending` / `approved` / `rejected`
- `deliverables.status`: `pending` / `approved` / `rejected`
- 완료 = 결과물 `approved`. 시스템 내 별도 "완료" 상태 없음

### 5-2. 완료 판정 (AND 매칭 대응)
`campaigns.channel_match` 기반:
- `or`: 선택 채널 중 1개 결과물 승인 → 완료
- `and`: 선택 채널 각각 결과물 승인 → 완료 (채널별 개별 결과물 필요)

> 현재 `channel_match='and'` 는 표시 용도로만 사용 중이며 완료 판정 로직은 미구현. 이번 릴리스에서 신규 구현.

### 5-3. 되돌리기 (신청·결과물 공통)
- 관리자 되돌리기 허용
- 인플루언서가 이미 확인한 건은 **강력 경고 모달** 후 진행 가능
- 변경 이력은 DB에만 기록 (UI 미노출)
- 결과가 바뀌면 인플루언서 알림에 **"결과가 변경되었습니다"** 명시

### 5-4. 동시 편집
- 관리자 2명이 같은 건 동시 처리 시: 후순위 액션은 **"이미 처리됨" 에러로 엄격 차단** (낙관적 락 or updated_at 체크)

## 6. 캠페인 기한 필드 (타입별 분리)

기존: `deadline`(모집마감), `post_deadline`(게시마감)

추가:
| 필드 | monitor | gifting | visit |
|------|---------|---------|-------|
| `purchase_start` / `purchase_end` | 필수 (구매기간) | — | — |
| `visit_start` / `visit_end` | — | — | 필수 (방문가능기간) |
| `submission_end` (결과물 제출 마감) | 필수 | 필수 | 필수 |

- `submission_end` 초과 시 인플루언서 제출 **차단**
- 기존 `post_deadline` 의미 재정의 or 신규 컬럼 도입은 DB 설계 단계에서 결정

## 7. 이력 보존

### 7-1. 영수증
- 전 제출분 보존: 이미지·금액·구매일·제출일·상태·반려사유
- 관리자 검수 모달에 **타임라인**으로 표시

### 7-2. 게시물 URL
- **동일 URL 재제출**: URL은 중복 저장 X, 제출 날짜만 누적 기록
- **다른 URL 재제출**: 이력 분리 저장
- 이력 타임라인 동일 표시

### 7-3. 신청 비승인 → 결과물 처리
- 결과물은 **삭제하지 않음**, 이력 보존 (신청 비승인 상태만 유지)

---

## 8. 반려 사유 입력 (관리자)

- **템플릿 + 자유 입력 혼합**, 필수
- 템플릿 후보 (기준 데이터화):
  - PR 태그 누락(#PR/#広告/#プロモーション)
  - 이미지 품질 부족 (흐림/식별 불가)
  - 금액/구매일 확인 불가
  - 게시물 삭제·비공개 상태
  - 필수 해시태그·멘션 누락
  - 기타 (자유 입력)

## 9. 제약 & 엣지 케이스 정책

| 케이스 | 정책 |
|------|------|
| 승인 후 게시물 SNS 삭제 | **신뢰 기반**, 시스템 감지 없음 (수동 문의로만 처리) |
| 신청자 존재 시 캠페인 타입 변경 | `campaign_admin` 이상만 허용 + 강력 경고 모달 |
| 영수증 금액 상한 초과 | 현재 경고 없음 (추후 검토) |
| 영수증 + URL 동시 필요 캠페인 | 이번 범위 제외 (타입은 배타적) |
| 지원 안 하는 URL | 드롭다운 수동 채널 선택 fallback |
| 기한 초과 제출 시도 | 차단 |

## 12. 이번 기획 범위 (1차 릴리스 제안)

**포함**
1. 결과물 관리 페인 (검수 몰아처리)
2. 캠페인 진행현황 탭 (캠페인 단위 전체 흐름)
3. 신청관리 pending 기본필터 전환
4. 인플루언서 타입별 결과물 제출 폼 + 복수 URL
5. 반려 사유 템플릿+자유입력, 이력 타임라인
6. 3중 알림(배너+배지+알림섹션) + 자동 읽음
7. 캠페인 기한 필드 확장 (타입별)
8. OT 수동 체크박스 + `oriented_at`

**제외 (후속)**
- OT 시트 이메일 자동 발송
- 게시물 URL 인앱 임베드
- PR 태그 자동 검출
- 정산 연동 (paid 상태)
- 반려/변경 이메일 알림
