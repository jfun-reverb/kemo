## 배경·목적

응모건 메시지 기능(인플루언서↔관리자)이 2026-05-28 운영서버에 출시된 직후,
운영 데이터베이스에 쌓인 **인플루언서 발신 문의 75건(2026-05-28~05-29, 이틀치)**을 분석.
어떤 질문이 많은지 유형화하여 추후 **자동응답(FAQ) 보강 + 운영 정책 결정**의 입력 자료로 삼는다.

## 현재 상태 (2026-05-29 기준)

### 운영 배포 상태 (git origin/main 확인)
- **응모건 메시지** — 운영 배포됨 (`messaging.js`/`admin-messaging.js` + 빌드 산출물 `page-messages`/`adminPane-messages`/`get_application_messages`)
- **자동응답(FAQ) 게이트** — 운영 배포됨 (`faq_nodes`/`record_faq_interaction`/「よくある質問」 버튼, 마이그레이션 146 포함). 시드 31노드
- 즉 **인플루언서는 이미 FAQ 게이트가 있는 문의 창구를 사용 중**

### 이 분석의 핵심 전제
- FAQ 게이트(문의 입력 중 유사 FAQ 제안 + 확인 시트)가 **이미 작동 중인데도 이 75건이 관리자에게 도달**했다
  → ① FAQ에 해당 답이 없거나 ② 있어도 인플이 「解決しました」를 안 누르고 「それでもお問い合わせ」로 넘어옴
  → **현재 FAQ 31노드의 빈 구멍을 실데이터로 드러낸 자료**

### 데이터 출처
- `application_messages` where `sender_kind='influencer'` and `hidden_by_admin_at is null` and `self_withdrawn_at is null`
- 75건 전체 노출 / 74개 응모건 / 가장 오래된 2026-05-28 · 가장 최근 2026-05-29

## 실측 유형별 분포 (75건)

| 순위 | 질문 유형 | 건수(대략) | 비중 |
|---|---|---|---|
| 1 | 상품 미배송·배송지연 + 마감 임박 | ~33 | 44% |
| 2 | 환급(返金) 언제 되나 | ~19 | 25% |
| 3 | 「이미지 제공형」 案件 가이드 부재 | ~8 | 11% |
| 4 | 구매 불가 (기한 경과·재고 소진·가격 2배 불일치) | ~6 | 8% |
| 5 | 심사·낙선 상태 혼란 (「落選なら審査中から消して」) | ~5 | 7% |
| 6 | 결과물 제출 오류·수정 요청 | ~3 | 4% |

> 주의: 「いつ返金されますか？」는 거의 동일 문구로 18건 이상 — 특정 리뷰어(구매형) 캠페인에 집중.
> 미배송 문의도 동일 문구 쌍이 많다. **일부는 개별 질문이 아니라 "한 캠페인의 배송·환급 사고에 여러 인플이 동시 반응"한 것.**

### 유형별 대표 원문(취지만, 개인정보 제외)
1. **미배송+마감**: 「商品がまだ届きません。5/31提出締切になっていますがどうしたら良いですか？」 / 「お品物が届いておらず、期限までの投稿が難しい。配送状況教えていただけますか？」 / 「伝票番号を教えていただくことは可能でしょうか？」
2. **환급**: 「いつ返金されますか？」(다수 동일)
3. **이미지 제공형**: 「商品提供なしで画像を掲載とのことですが、ご指定の画像はどちらから確認できますか？」 / 「提供された画像を使用して投稿…どの画像を使用すればよいか記載がない」
4. **구매 불가**: 「購入期限過ぎています」 / 「在庫切れのためキャンセルされていました」 / 「Qoo10と金額が2倍違うので購入できません」
5. **심사·낙선**: 「落選したのであれば審査中から消していただきたいです」 / 「審査はいつ終わりますか？」
6. **제출 오류**: 「誤って別の商品のスクリーンショットをアップロード…修正できない」 / 「以前提出したのに提出していませんとメール…再度提出したがどうなっているか」

## 핵심 발견

### ① "FAQ로 풀 문제"와 "운영 사고"가 섞여 있다 — 반드시 분리
| 영역 | FAQ·안내로 예방 가능 | 운영 처리 필요 (안내문으로 안 풀림) |
|---|---|---|
| 배송 | "상품 미도착/마감 내 제출 불가 시" 절차 안내 | 실제 배송 누락·송장 오등록 (Qoo10 장문 사례) |
| 환급 | "환급 시점·방식" 표준 안내 | 특정 캠페인 환급 지연 자체 처리 |
| 이미지 | "이미지 제공형 案件 진행법" 안내 | **이미지를 실제 전달하는 경로가 없음(운영 프로세스 구멍)** |

### ② 앱 메시지가 LINE 백로그를 흡수 중
- 다수 메시지가 「LINEで問い合わせても返信がない」로 시작
- 메모리 「LINE 소통 최소화 + 앱 단일 창구 통합」 방향이 맞다는 **실데이터 확인**
- 사양서 `docs/specs/2026-05-21-feedback-channel-roles.md` 와 연결

## FAQ 개선 포인트 (현재 31노드 대비 빈 구멍)

1. **「상품이 아직 안 왔어요 / 마감까지 못 받을 것 같아요」** (최대 구멍)
   - 필요한 답: ①배송 조회 방법 ②마감 자동 유예/연장 정책 ③언제까지 기다리면 되는지
   - **선결 조건: 마감 정책(자동 연장 여부) 자체를 먼저 정해야** 제대로 된 답을 쓸 수 있음 → 정책 결정 항목
2. **「환급은 언제, 어떻게 되나요」** (리뷰어 구매형 전용) — 표준 시점·방식 명시
3. **「이미지 제공형 案件은 어떻게 진행하나요」** — FAQ 추가 + 운영상 이미지 전달 경로 정립 동반 필수
4. **「떨어지면 심사중에서 빼주세요」** — 심사 결과(낙선) 통지·상태 표시 개선 (FAQ보다 UX 영역)
5. **「가격이 2배 달라 구매 못 해요」** — 캠페인 등록 가격 vs 실제 마켓 가격 불일치 (운영 검수 영역)
