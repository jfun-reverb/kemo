influencers 테이블에 컬럼 추가 (**보호자 컬럼 없음 — 단순화**):

## 4. 기존 회원 소급 게이트
> ⚠️ **§0-1 ①로 갈음(2026-06-15)**: "로그인 즉시 닫을 수 없는 모달"은 **폐기**. 게이트는 **응모 버튼 누를 때** 발동(로그인·둘러보기는 자유). `birthdate` NULL이거나 18세 미만이면 응모 모달 진입 시 생년월일·성별 입력 요구 → 미입력·미달이면 응모 차단. 진입점은 `app.js` init이 아니라 `application.js`의 응모 시작 함수(`handleFloatApply` 직전, 현재 missing 검증 라인 484~502 인근). 아래 본문은 "로그인 즉시 모달" 가정이라 효력 없음 — 응모 시점 게이트로 읽을 것.
- **(폐기된 원안)** 모달 게이트 + 서버 차단 병행: 로그인/세션 복원 후(`app.js` init) `birthdate` NULL이면 닫을 수 없는 모달로 생년월일·성별 입력 요구
- 차단 범위 = **응모(신청)만 차단, 둘러보기 허용**(이탈 최소)
- **18세 미만 판명 시**: 강제 탈퇴 없음. 계정·둘러보기 유지하되 **응모만 차단**(서버). 안내 문구로 「18歳になるまで応募はできません」. 18세 도달 시 자동 해제(서버가 birthdate로 매번 판정하므로 별도 처리 불필요)
- 엣지: 다중 탭 동기화, 입력 도중 새로고침 시 모달 재표시, **관리자 계정 제외**(admins는 influencers 게이트 대상 아님)
