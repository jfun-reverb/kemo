# 노션 실무자 가이드 — 스크린샷 자동 캡처 파이프라인 (트랙 B 핸드오프)

**작성일:** 2026-06-18
**작성:** 고문 세션
**받는 쪽:** 개발 세션
**관련:** 노션 "실무자 운영 가이드" 13개 페이지 새 스타일 재작성(2026-06-18 완료), 자동 작업 "reverb-jp daily deploy digest" 정의 현행화

---

## 배경

노션 "실무자 운영 가이드" 하위 13개 페이지를 "쉽고 시각적인" 새 스타일로 전면 재작성하면서, 각 페이지에 **스크린샷 자리(약 31곳)**를 회색 콜아웃으로 배치했다. 형식:

```
<callout icon="📸" color="gray_bg">
	**[화면 N] (무슨 화면)** · *자동 캡처 대상(개발서버)*
	(그 화면에 무엇이 보여야 하는지 한 줄)
</callout>
```

사용자 결정(2026-06-18): **핵심 화면만 자동 캡처 + 나머지는 사람이 수동 첨부.**

이 문서는 그 "자동 캡처" 트랙을 개발 세션이 구현할 수 있게 정리한 핸드오프다. (노션 가이드 텍스트·스타일 작업은 고문이 이미 완료 — 이 문서는 **그림 채우기**만 다룬다.)

---

## 목표

스크린샷 자리 중 "핵심 화면"에, **개발서버 관리자 화면을 캡처한 이미지**를 노션 이미지 블록으로 채워 넣는다.

---

## 보안 제약 (반드시 준수)

- **캡처는 개발서버(dev.globalreverb.com/admin/)에서만.** 운영(globalreverb.com)에는 실제 인플루언서 개인정보가 떠 있어, 캡처해 노션(외부 문서)에 올리면 개인정보가 박힌다. 개발서버는 더미 데이터라 안전.
- 캡처 전 화면에 떠 있는 데이터가 더미인지 한 번 확인. 혹시라도 실데이터가 보이면 캡처 중단.

---

## 기술 방식 (제안 — 개발 세션이 최종 결정)

노션 이미지 블록은 "인터넷에서 열리는 이미지 URL"만 임베드할 수 있다(`![캡션](URL)`). 로컬 캡처 파일 직접 업로드 경로가 없다. 따라서:

1. **캡처**: Playwright로 개발서버 관리자 화면 로그인 후 해당 페인/모달 캡처.
   - ⚠️ Playwright는 크롬 1개를 공유하는 **단일 자원**이다. 다른 세션이 qa-test 중이면 실행 금지(먼저 돌던 게 멈춘다). `.claude/rules/multi-session.md` 「Playwright 단일 자원」 참조.
2. **호스팅**: 캡처 이미지를 외부에서 열리는 URL로.
   - 1순위 후보: **Supabase Storage 공개 버킷**(프로젝트가 이미 Supabase를 씀). 예: `notion-guide-shots/` 버킷 신설 + 공개 읽기 정책. 캡처 → 업로드 → 공개 URL 획득.
   - 대안: GitHub raw(레포에 `docs/notion-shots/` 커밋), 기타 이미지 호스팅.
3. **삽입**: 노션 `update-page`(update_content)로 해당 `[화면 N]` 콜아웃 **아래에** 이미지 블록 추가. 기존 콜아웃(자리표시)은 남겨도 되고, 이미지로 대체해도 됨(사용자 선호 확인).

## 스크린샷 자리 목록 (페이지별 [화면 N])

| 페이지 | 화면 자리 |
|---|---|
| 신청 관리 | ① 신청 목록 전체 · ② 인플루언서 상세 모달 |
| 대시보드 | ① 대시보드 전체 · ② 차트 영역(가입추이·도도부현·완성률) · ③ 최근 신청 테이블 |
| 캠페인 관리 | ① 캠페인 목록 전체 · ② 결과물 엑셀 파일 내용 · ③ 상태 드롭다운+「?」 도움말 |
| 캠페인 등록/편집 폼 | ① 폼 전체(4섹션) · ② 우측 실시간 미리보기 사이드바 |
| 참여방법 번들 | ① 기준데이터 참여방법 탭 묶음 목록 · ② 새 번들 등록 모달 |
| 결과물 관리 | ① 결과물 목록 전체 · ② 결과물 검수 상세 모달 (+필요 시 1) |
| 알림 시스템 | ① 인플루언서 알림 모달 · ② 관리자 메일받기 설정 |
| 인플루언서 관리 | ① 인플루언서 목록 전체 · ② 인플루언서 상세 프로필 |
| 기준데이터 관리 | ① 기준데이터 전체(8개 탭) · ② 참여방법 번들 편집 |
| 관리자 계정 | ① 관리자 계정 목록 · ② 메일 수신 설정 창 |
| 내 계정 | ① 내 계정 화면 |
| 공지사항 시스템 | ① 공지 목록 · ② 공지 등록/수정 창 |
| 광고주 신청(브랜드 서베이) | ① 신청 목록 · ② 신청 상세 모달 · ③ 캠페인 진행현황 요약 카드 |

> 합계 약 31곳. "핵심 화면" 우선순위는 사용자/개발 세션이 정한다. 각 페이지의 콜아웃에 "무엇이 보여야 하는지" 한 줄이 이미 적혀 있으니, 캡처 시 그 설명을 기준으로.

---

## 미해결 결정 사항 (개발 세션이 사용자와 확인)

1. **호스팅 위치**: Supabase Storage 공개 버킷 vs GitHub raw vs 기타. (개인정보 없는 더미 화면 캡처라 공개 버킷도 무방하나, 버킷 신설·정책은 사용자 확인)
2. **자리표시 처리**: 이미지를 콜아웃 아래에 추가할지, 콜아웃을 이미지로 대체할지.
3. **"핵심 화면" 범위**: 31곳 전부 자동 vs 자주 보는 N곳만 자동.
4. **갱신 주기**: 화면이 바뀌면 캡처도 낡는다 → 1회성 vs 주기적 재캡처. (자동 작업이 "검토 필요"를 보고하므로, 화면 크게 바뀐 페이지만 재캡처하는 방식 권장)
