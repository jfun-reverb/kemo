## 1. 배경 / 문제

### 현재 상태
- TERMS_kr/ja, PRIVACY_kr/ja 4종이 git history 안에서 매번 직접 수정됨
- 부칙(개정 이력)이 bullet list 형태로 누적 — 시간 갈수록 가독성 떨어짐
- 인플이 옛 약관 버전을 확인할 방법이 없음 (git history 만 보관, 인플은 접근 불가)
- 약관 변경 통지가 부칙 한 줄 추가로만 처리 — 인플이 변경 사실을 인지하기 어려움
- 기능 출시와 약관 갱신 일정이 어긋남 — 빠른 기능 배포 → 사후 약관 정비 (technical debt 누적, 2026-05-28 사용자 명시)

### 문제
- 한국 PIPA·일본 APPI 「개정 안내」 의무 충족 채널이 약하다 — 부칙 추가만으로는 「인플이 알 수 있는 방법으로 공지」 충분 여부 불확실
- 변경 전·후 내용 비교 채널이 전혀 없음 — 인플이 「뭐가 어떻게 바뀌었는지」 모른 채 그냥 「변경된 약관에 동의한 것으로 간주」됨
- 약관 개정이 누적되면 「내가 동의했던 시점의 약관 본문」을 인플이 추적 불가
- 운영 거버넌스 부재 — 기능 PR 머지 시점에 약관 영향 점검이 일관되게 적용되지 않음

### 목표
- **인플 친화도 ↑**: 변경 사항을 명확히 알리는 메일 + 「이전 버전 보기」 + 비교 표
- **법적 안전성 ↑**: 한국 PIPA·일본 APPI 의무 명확히 충족 (중대 변경 30일 / 일반 변경 7일 차등 통지)
- **운영 거버넌스 전환**: 기능 도입일정 = 약관 명시·동의 절차 완료 일정 동기화 (점진 전환, 2026-05-28 사용자 결정)

### 5-2. Phase 2 — 버전 분리 보관 + 인플 「이전 버전 보기」

#### 5-2-1. 디렉토리 구조

```
docs/
├── TERMS_kr.md           ← 현재 시행 중인 최신 버전
├── TERMS_ja.md
├── PRIVACY_kr.md
├── PRIVACY_ja.md
└── legacy/               ← 신설
    ├── TERMS_kr_v1.md    ← 제1차 개정 직전 시점 본문 복사
    ├── TERMS_ja_v1.md
    ├── PRIVACY_kr_v1.md
    ├── PRIVACY_ja_v1.md
    ├── TERMS_kr_v2.md    ← 제2차 개정 직전 시점 (개정 누적 시 추가)
    └── ...
```

#### 5-2-2. legacy 파일 관리 패턴

- 약관 개정 작업 직전: 현재 `TERMS_kr.md` 를 `docs/legacy/TERMS_kr_v{현재_회차}.md` 로 복사
- 그 후 본문 수정
- 부칙 「개정 이력」 표에 새 행 추가 — 「제N차」 라벨이 `legacy/TERMS_kr_v{N-1}.md` 와 매칭됨 (직전 버전 = N-1차)
- 4종 동시 처리 (`.claude/rules/policy.md` 한·일 양언어 동시 갱신 영구 규칙)

#### 5-2-3. legal.js 확장 — 「이전 버전 보기」

```javascript
// 기존
const LEGAL_DOCS = {
  terms:   { ko: 'docs/TERMS_kr.md',   ja: 'docs/TERMS_ja.md',   ... },
  privacy: { ko: 'docs/PRIVACY_kr.md', ja: 'docs/PRIVACY_ja.md', ... }
};

// 확장
function legacyDocPath(kind, locale, version) {
  // kind: 'terms'|'privacy', locale: 'ko'|'ja', version: 1, 2, ...
  const prefix = kind === 'terms' ? 'TERMS' : 'PRIVACY';
  const lang = locale === 'ko' ? 'kr' : 'ja';
  return `docs/legacy/${prefix}_${lang}_v${version}.md`;
}
```

#### 5-2-4. 인플 UI — 「이전 버전 보기」

`#page-legal` 하단에 토글:
- 「이전 버전 보기 ▼」 버튼 클릭 시 펼침
- 버전 목록 표시: 제1차 (시행 2026-05-01~2026-05-27) / 제2차 (시행 2026-05-28~) ...
- 각 버전 클릭 시 해당 `legacy/` 파일 fetch + 렌더 (현재와 동일 마크다운 렌더러)
- 상단에 「**이 버전은 N월 N일 ~ N월 N일 시행되었던 약관입니다**」 배지 표시 (인플이 현재 시행 약관과 헷갈리지 않게)

## 6. 약관 본체 신규 조항 — TERMS 제25조

### TERMS_kr 신규 조항

```markdown
### 제25조 (약관의 개정)

1. 회사는 본 약관을 개정할 수 있으며, 개정 시에는 적용 일자 및 개정 사유를 명시하여 현행 약관과 함께 다음 각 호의 방법으로 회원에게 사전 공지합니다:
   - 회사 서비스 내 공지사항 게시
   - 회원이 등록한 이메일 주소로 통지
   - 회사 앱 내 알림

2. 회사는 본 약관을 개정할 경우 그 개정 시행일 7일 전부터 공지합니다. 다만 회원의 권리·의무에 중대한 변경이 있는 경우(개인정보 수집 항목·이용 목적·제3자 제공·국외 이전·관리자 권한의 신설 등)에는 시행일 30일 전부터 공지합니다.

3. 회원은 개정된 약관에 동의하지 않을 권리가 있으며, 이 경우 시행일 전까지 회사에 거부 의사를 표시하고 본 서비스 이용 계약을 해지할 수 있습니다.

4. 회원이 본 조 제2항에 따른 공지에 대해 시행일 전까지 별도의 거부 의사를 표시하지 않은 경우, 개정된 약관에 동의한 것으로 간주합니다.

5. 회원은 회사 앱 내 「이전 버전 보기」 메뉴를 통해 과거 시행되었던 약관 본문을 언제든지 열람할 수 있습니다.
```

### TERMS_ja 동등 조항

```markdown
### 第25条 (規約の改定)

1. 当社は本規約を改定することができ、改定する際には適用日および改定理由を明示し、現行規約と共に次の各号の方法により会員に事前公告します:
   - 当社サービス内のお知らせ掲載
   - 会員が登録したメールアドレスへの通知
   - 当社アプリ内通知

2. 当社が本規約を改定する場合、施行日の7日前から公告します。ただし、会員の権利・義務に重大な変更がある場合(個人情報の収集項目・利用目的・第三者提供・越境移転・管理者権限の新設など)には、施行日の30日前から公告します。

3. 会員は改定された規約に同意しない権利を有し、その場合、施行日までに当社に拒否の意思表示を行い、本サービスの利用契約を解除することができます。

4. 会員が本条第2項の公告に対し、施行日までに別途の拒否の意思表示を行わない場合、改定された規約に同意したものとみなします。

5. 会員は当社アプリ内「過去バージョンを見る」メニューより、過去に施行されていた規約本文をいつでも閲覧することができます。
```
