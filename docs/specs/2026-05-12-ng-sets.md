## 2. 데이터베이스 스키마 변경

### 2-1. 마이그레이션 파일

**파일명**: `supabase/migrations/{다음번호}_create_ng_sets.sql`

> 다음 마이그레이션 번호는 작업 시작 시점에 `ls supabase/migrations/ | tail -5`로 확인. 현재 dev에 104까지 존재(응모 취소 PR-A 점유). 응모 취소 PR-D 알림 마이그레이션도 추가될 예정이라 NG 본 마이그레이션은 **둘 다 머지 후 다음 번호**.

### 2-2. `ng_sets` 신규 테이블

```sql
CREATE TABLE public.ng_sets (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name_ko         text NOT NULL,
  name_ja         text NOT NULL,
  recruit_types   text[] NOT NULL DEFAULT '{}'::text[],
  items           jsonb NOT NULL DEFAULT '[]'::jsonb,
  sort_order      integer NOT NULL DEFAULT 0,
  active          boolean NOT NULL DEFAULT true,
  created_at      timestamptz NOT NULL DEFAULT now(),
  created_by      uuid REFERENCES auth.users(id) ON DELETE SET NULL,
  updated_at      timestamptz NOT NULL DEFAULT now(),
  updated_by      uuid REFERENCES auth.users(id) ON DELETE SET NULL
);

CREATE INDEX ng_sets_active_sort_idx ON public.ng_sets (active, sort_order);
```

**행 단위 보안 정책(RLS)** — 주의사항 번들 정책 그대로 미러:
- SELECT: 관리자만 (`is_admin()`)
- INSERT/UPDATE/DELETE: campaign_admin 이상

> 인플루언서는 `ng_sets`에 직접 접근하지 않음. `campaigns.ng_items` 스냅샷만 SELECT.

### 2-3. `campaigns` 컬럼 추가

```sql
ALTER TABLE public.campaigns
  ADD COLUMN ng_set_id  uuid NULL REFERENCES public.ng_sets(id) ON DELETE SET NULL,
  ADD COLUMN ng_items   jsonb NOT NULL DEFAULT '[]'::jsonb;
```

- `ng_set_id`: 번들 원본 추적용 (번들 hard delete 시 SET NULL로 격리)
- `ng_items`: 캠페인 저장 시 번들 items 통째 복사한 스냅샷
- 기존 `campaigns.ng text` 컬럼은 **유지** (Q1 결정) — `COMMENT ON COLUMN ... IS 'DEPRECATED (2026-05-12). 신규 ng_items jsonb로 전환. 1주 관찰 후 별도 마이그레이션으로 DROP 검토.'` 표시

### 2-4. 아이템 스키마 (주의사항과 동일)

```typescript
type NgItem = {
  html_ko: string;          // 한국어 차소 텍스트 (DOMPurify 통과 HTML 허용 — bold/italic/underline/strike/link + 이미지)
  html_ja: string;          // 일본어 차소 텍스트
  link_url?: string;        // 선택 링크 (item 단위)
  link_label_ko?: string;
  link_label_ja?: string;
  text_after_ko?: string;   // 링크 뒤 잇는 텍스트
  text_after_ja?: string;
};
```

XSS 방어: 저장+렌더 이중 sanitize (DOMPurify) — 주의사항 함수 `sanitizeCautionItems` / `renderCautionItems` 와 동일한 헬퍼를 `sanitizeNgItems` / `renderNgItems` 로 추가.

> **2026-05-12 추가 확장**: `docs/specs/2026-05-12-rich-editor-image-upload.md` 사양으로 미니 에디터에 **이미지 삽입 기능**이 추가됨. NG 항목의 `html_ko`/`html_ja`도 자동으로 이미지를 포함할 수 있게 됨. DOMPurify allowed 태그에 `img/p/br` 추가, `src`는 Supabase Storage 도메인 + https 화이트리스트. 본 NG 사양 PR-B가 본 미니 에디터 강화 PR 머지 후 진행되면 별도 코드 변경 없이 자동 적용.

## 3. 비즈니스 룰 매트릭스

### 3-1. 캠페인 저장 시점 동작

| 시나리오 | 동작 |
|---|---|
| 신규 캠페인 등록 + 번들 선택 | 번들 items를 `campaigns.ng_items`에 통째 복사 + `ng_set_id` 저장 |
| 신규 캠페인 등록 + 번들 미선택 | 사용자가 인라인으로 직접 입력한 항목들을 `ng_items`에 저장 + `ng_set_id=NULL` |
| 기존 캠페인 편집 + 「번들 다시 불러오기」 | 현재 번들의 items로 `ng_items` 통째 덮어씀 (수동 동기화) |
| 기존 캠페인 편집 + 인라인 수정 | 사용자가 편집한 결과를 `ng_items`에 저장. `ng_set_id`는 그대로 (또는 NULL로 변경 가능) |
| 번들 본문 수정 후 캠페인 저장 안 함 | 캠페인 `ng_items`는 변동 없음 (스냅샷 격리) |
| 캠페인 `status='closed'` 후 NG 변경 시도 | DB 트리거가 차단 (RAISE EXCEPTION) |
| 번들 hard delete | `ON DELETE SET NULL`로 `ng_set_id`만 끊고 `ng_items` 스냅샷은 보존 |

### 3-2. 인플루언서 화면 분기

| `ng_items` 상태 | `campaigns.ng` (legacy) | 표시 |
|---|---|---|
| 비어있지 않음 | N/A | `ng_items` 항목 리스트 렌더 (우선) |
| 비어있음 | 비어있지 않음 (legacy) | `richHtml(camp.ng)` 폴백 렌더 (현재 동작) |
| 비어있음 | 비어있음 | NG 섹션 자체 숨김 (현재 동작) |

→ 인플루언서 캠페인 상세에서 자연스러운 폴백. legacy 캠페인은 운영자가 편집 시점에 점진적으로 마이그레이션.
