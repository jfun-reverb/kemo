# 코드맵 (CODEMAPS) — REVERB JP 현 상태 지도

> 신규 기능·수정 작업에 착수하기 전, **해당 영역 코드맵 1장만 읽으면 현 상태를 파악**하도록 만든 인덱스다. 9,545줄짜리 `admin.js`나 2,243줄짜리 명세를 매번 처음부터 훑지 않기 위한 것.

## ⚠️ 이 문서의 위치

- **작업 시작점 지도**이지 검증된 정본(source of truth)이 아니다. 코드 자체와 `docs/FEATURE_SPEC.md`가 정본.
- 최초 생성은 자동 탐색 에이전트(병렬)로 만들었기 때문에 **줄 번호에 오차가 있을 수 있다.** 실제 수정 전에는 함수명으로 한 번 더 grep 확인할 것.
- 줄 번호보다 **"어느 파일·어느 함수·어느 기능 그룹"** 의 위치 정보를 신뢰하라.

## 작업 워크플로 (권장)

1. **현 상태 파악** — 착수할 작업이 걸치는 영역의 코드맵을 먼저 읽는다. 여러 영역에 걸치면 탐색 에이전트(Explore)를 영역별로 동시에 띄워 병렬로 파악한다.
2. **기획** — 파악한 현 상태를 바탕으로 `reverb-planner` 호출.
3. **개발** — 구현 + 빌드 + `reverb-reviewer`.

## 유지 규칙 (중요 — 안 지키면 3주 뒤 신뢰 못 하는 문서가 됨)

- **기능 커밋이 해당 영역의 함수를 추가·이동·삭제하면, 같은 작업에서 코드맵의 해당 항목도 갱신**한다. (`docs/FEATURE_SPEC.md` 갱신 의무와 같은 선상 — `.claude/rules/docs-tracking.md` 참조)
- 줄 번호 전수 갱신은 부담이므로, **함수의 추가·삭제·이름 변경·기능 그룹 이동만 반영**하면 충분하다. 단순 내부 수정으로 줄이 밀린 것은 갱신 대상 아님.
- 영역이 커져 코드맵이 비대해지면(예: `admin.js` 분할 시) 코드맵도 함께 쪼갠다.
